General Information About This Project


The Project Components

I think I will use this space to talk a bit about components in Angular and what components I created and what are within them.

First, my Angular-App has a nav component, which is (you guessed it) for navigations through different components. This component will always be visible but based on what you click, the rest of the page will be a different component.

By default, I have my home as the start of the application, which was hard for me to figure out content for. Not much went into it besides some SASS and BootStrap.

As of right now, I have another component, which is meant to be one of eventually several projects, weather. This component imports DataService which is used for the REST when getting returned JSON from the National Weather Service (NWS). The import is done in this components TypeScript file. Then there is the FormsModule import that is done through the app.module.ts file which allows me to do Two-Way data binding between form-controls, such as inputs, and modules. Two-Way data binding and how it is easily achieved is definayely the most interesting thing I have come across so far with angular, that along with components.

What Went Into This Project

For starters, the main purpose of this was to gain some exposure with Angular. I decided to work with Angular 7 because it is the latest version. However, by the time anyone is looking at this, it may not be. So here is probably the best place to see what is the latest version as well as predictions on future releases.


However, when getting started with Angular I had a few other things that I needed to pick up to really get things started.


  • BootStrap 4.2.1
  • SASS
  • TypeScript


Here are a couple of other topics/techniques relevant to this that aren't exactly new to me, but are important to note. There may be more but these are key for how the projects I have or will create are going to work.


  • API Get Requests
  • JSON Parsing
  • Data Binding
  • Document Object Model (DOM)

Back to speaking about Angular, it wasn't really difficult to understand what was going on. Through some dedication and persistence, anyone could learn how to use Angular and pick up what I did. What I am doing to display these skills aren't meant to be difficult, just a way to pick up the skills and then show them on my GitHub.

That being said, I have one project as of now, which is the Weather App, which does exactly what it says. There is a form that allows users to enter a longitude and latitude of their desire and on submission sends a RESTful request to the National Weather Service (NWS) and displays (in a semi-nice way) weather information of the location asked for.
Note: There is not much error-hendling present in the app currently because I am focusing on getting stuff to work not foolproofing an app. Also, I have been trying to work on making nicer layouts, and may or may not be succeading.
